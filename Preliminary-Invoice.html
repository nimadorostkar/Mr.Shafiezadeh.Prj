The engagement

Design & development of a personal knowledge platform

This preliminary proposal outlines two engagement paths — a lean Economic build first, and a complete Professional build — described by scope and timeline. The two cases, explained by timing

1 · Economic

≈ 17 weeks · 4 months (solo, ~5 days/week)

A lean path to a working product. It compresses design to the essentials and ships the core backend + frontend, key integrations, and a basic deploy — trading extensive test automation, a formal design system, and full ops tooling for speed. Ideal to validate the idea in real use first.

  • Core content graph & three-policy backend (Django/DRF)
  • Frontend for the primary screens
  • Essential integrations + code review + short support window
  • Docker + VPS deploy; lighter testing & ops
  • Best to launch fast, prove daily use, then grow into Option 2.

2 · Professional

≈ 25–26 weeks · 6 months (solo, ~5 days/week)

The complete, durable build. Time goes up front into discovery, UX research, a real design system, and formal system design, then through a full testing suite, CI/CD, monitoring, and an operations window — depth now for an extensible, low-maintenance platform for years, matching the project's "quality over speed, longevity is a feature" principles.

  • Full discovery → strategy → IA → UX → UI design track
  • Design system, component library & accessibility
  • Formal DB / API / security design before coding
  • Complete automated testing (unit → E2E, performance, security)
  • CI/CD, release strategy, monitoring & observability
  • Best when this is the long-term home of your digital presence.

Both estimates assume a single practitioner working ~5 days/week with timely client feedback; elapsed calendar time may extend with review cycles. The scopes are additive — starting with Option 1 and later adding the remaining Professional deliverables reaches the same end state.
